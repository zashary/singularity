Singularity

Containers without the container!

Singularity creates a new application distribution paradigm by utilizing a combination of both packaging and container principals while maintaining standard command line interface work flows.

One of the primary design goals was to lower the bar to container based application portability and distribution without the traditional pitfalls of containers when used in this way (e.g. privilege escalation within the containers which lead to network access issues and thus VPN segregation as well as management of an entire operating system load). While there are very strong use cases for containers (and virtual machines), application portability is not necessarily a good use of these technologies. Singularity addresses this directly.

In the above video, you will see how singularity can be used in several very simple examples. These particular examples are used to demonstrate how Singularity operates and it should be mentioned that a more typical usage could include custom optimized binaries, specific data or input requirements, or an entire complicated work flow.
